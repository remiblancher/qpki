# QPKI Quality Assurance

> Quality documentation following ISO/IEC 25010 (Software Quality) and ISO/IEC 29119 (Testing).

## Overview

| ISO 25010 Characteristic | Documentation | Source |
|--------------------------|---------------|--------|
| **Functional Suitability** | [Test Strategy](testing/STRATEGY.md) | Manual |
| **Reliability** | [Test Catalog](testing/CATALOG.md) | Generated |
| **Security** | [FIPS Compliance](compliance/FIPS.md) | Generated |
| **Compatibility** | [RFC Compliance](compliance/RFC.md) | Generated |
| **Interoperability** | [Interop Matrix](compliance/INTEROP.md) | Manual |

## Structure

```
docs/quality/
├── testing/                    # ISO 29119 - Test documentation
│   ├── STRATEGY.md             # Testing philosophy and approach
│   ├── CATALOG.md              # Exhaustive test catalog (generated)
│   └── COVERAGE.md             # Coverage report (generated by CI)
│
├── compliance/                 # Standards compliance
│   ├── FIPS.md                 # FIPS 203/204/205 (generated)
│   ├── RFC.md                  # RFC 5280, 5652, etc. (generated)
│   └── INTEROP.md              # Interoperability matrix
│
└── security/                   # Security
    └── THREAT-MODEL.md         # Threat model (coming soon)
```

## Single Source of Truth

Generated documentation comes from machine-readable specifications:

```
specs/compliance/standards-matrix.yaml  ──>  docs/quality/compliance/FIPS.md
                                        ──>  docs/quality/compliance/RFC.md

specs/tests/test-catalog.yaml           ──>  docs/quality/testing/CATALOG.md
```

**Rule**: Never edit generated files directly. Modify the source in `specs/` and regenerate.

## Regenerate Documentation

```bash
# Regenerate all quality documentation
make quality-docs

# Or individually
./scripts/generate-compliance-docs.sh    # Compliance docs
./scripts/generate-test-catalog-docs.sh  # Test catalog
```

## Quality Metrics

| Metric | Value | Target | Command |
